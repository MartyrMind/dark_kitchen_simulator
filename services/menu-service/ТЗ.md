# ТЗ для агента: Этап 3 — Menu Service

> Кодировка файла: UTF-8 with BOM.  
> Если читаешь файл из Python, используй `encoding="utf-8-sig"`.

---

## 0. Контекст

Проект — polyglot monorepo для микросервисной системы dark kitchen.

Уже должны быть реализованы предыдущие этапы:

```text
Этап 1:
  libs/python/dk-common
  общие settings/logging/correlation/health/errors

Этап 2:
  services/kitchen-service
  управление kitchens/stations без KDS
```

Этот этап — **Menu Service**.

Menu Service отвечает только за:

```text
- блюда;
- доступность блюд на конкретных кухнях;
- рецепты блюд в виде упорядоченных recipe steps.
```

Menu Service не должен создавать заказы, kitchen tasks, KDS tasks, Redis-сообщения или Mongo events.

---

## 1. Цель этапа

Реализовать `menu-service` без интеграции с Fulfillment Service.

К концу этапа должно быть возможно:

```text
1. Создать блюдо.
2. Получить список блюд.
3. Получить конкретное блюдо.
4. Добавить recipe steps к блюду.
5. Получить рецепт блюда в правильном порядке.
6. Установить доступность блюда для конкретной кухни.
7. Получить меню конкретной кухни с учетом доступности.
```

---

## 2. Scope этапа

На этом этапе нужно реализовать:

```text
1. FastAPI-приложение services/menu-service.
2. Подключение PostgreSQL.
3. SQLAlchemy async.
4. Alembic migrations.
5. Модели:
   - menu_items
   - kitchen_menu_availability
   - recipe_steps
6. Pydantic-схемы request/response.
7. API endpoints меню, рецептов и availability.
8. Подключение dk-common:
   - settings
   - logging
   - correlation middleware
   - health helper
   - common errors, если нужно
9. Базовые unit/component tests.
10. README для запуска menu-service.
```

---

## 3. Out of scope

На этом этапе НЕ делать:

```text
- Fulfillment Service;
- создание orders;
- создание kitchen_tasks;
- task_dependencies в Fulfillment Service;
- Redis Streams;
- MongoDB event log;
- Prometheus business metrics;
- KDS;
- Station Simulator;
- Go worker;
- HTTP-клиент к Kitchen Service;
- проверку существования kitchen_id через Kitchen Service.
```

Важно: `kitchen_id` в `menu-service` хранится как UUID-значение во внешней ссылке, но на этом этапе Menu Service не обязан проверять, существует ли такая кухня в Kitchen Service.

Проверка существования кухни будет на уровне Fulfillment Service или более поздней интеграции.

## 5. Зависимости и подключение `dk-common`

### 5.1. Важное правило

`dk-common` является отдельным installable Python package.

У `dk-common` может быть собственный `.venv` для разработки самой библиотеки, но `menu-service` не должен импортировать код из venv библиотеки.

`menu-service` должен иметь свой venv и устанавливать `dk-common` как обычную path dependency.

Запрещено:

```python
import sys
sys.path.append("../../libs/python/dk-common")
```

Запрещены относительные импорты из `libs` через `../../../`.

Правильный импорт после установки зависимости:

```python
from dk_common.logging import configure_logging
from dk_common.correlation import CorrelationIdMiddleware
from dk_common.health import build_health_response
from dk_common.settings import BaseServiceSettings
```

### 5.2. `pyproject.toml` menu-service

Пример `services/menu-service/pyproject.toml`:

```toml
[tool.poetry]
name = "menu-service"
version = "0.1.0"
description = "Menu Service for dark kitchen system"
authors = ["Dark Kitchen Team"]
packages = [{ include = "app" }]

[tool.poetry.dependencies]
python = "^3.11"

dk-common = { path = "../../libs/python/dk-common", develop = true }

fastapi = "^0.115.0"
uvicorn = { extras = ["standard"], version = "^0.30.0" }

sqlalchemy = { extras = ["asyncio"], version = "^2.0.0" }
asyncpg = "^0.29.0"
alembic = "^1.13.0"

pydantic = "^2.0.0"
pydantic-settings = "^2.0.0"

[tool.poetry.group.dev.dependencies]
pytest = "^8.0.0"
pytest-asyncio = "^0.23.0"
httpx = "^0.27.0"

[build-system]
requires = ["poetry-core"]
build-backend = "poetry.core.masonry.api"
```

### 5.3. Нужно ли дублировать зависимости `dk-common`

Нет.

Если зависимость используется только внутри `dk-common`, она должна быть объявлена в `libs/python/dk-common/pyproject.toml`.

Если зависимость напрямую используется кодом `menu-service`, она должна быть объявлена в `services/menu-service/pyproject.toml`.

Пример:

```text
menu-service напрямую использует FastAPI, SQLAlchemy, Alembic, asyncpg.
Значит они должны быть в pyproject.toml menu-service.

menu-service не должен напрямую использовать Loguru, если он вызывает только configure_logging().
Значит Loguru может оставаться зависимостью dk-common.
```

### 5.4. Docker / uv установка

Docker build context должен быть корнем репозитория, чтобы контейнер видел и сервис, и `libs/python/dk-common`.

Пример Dockerfile:

```dockerfile
FROM python:3.11-slim

ENV PYTHONDONTWRITEBYTECODE=1
ENV PYTHONUNBUFFERED=1

WORKDIR /repo

RUN pip install --no-cache-dir uv

COPY libs/python/dk-common /repo/libs/python/dk-common
COPY services/menu-service /repo/services/menu-service

RUN uv pip install --system /repo/libs/python/dk-common
RUN uv pip install --system /repo/services/menu-service

WORKDIR /repo/services/menu-service

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

Пример compose build:

```yaml
services:
  menu-service:
    build:
      context: ../..
      dockerfile: services/menu-service/Dockerfile
```

## 6. Settings

Создать `app/core/config.py`.

Использовать `BaseServiceSettings` из `dk-common`.

Пример:

```python
from functools import lru_cache

from dk_common.settings import BaseServiceSettings


class Settings(BaseServiceSettings):
    service_name: str = "menu-service"
    database_url: str = "postgresql+asyncpg://postgres:postgres@localhost:5432/menu_service"


@lru_cache
def get_settings() -> Settings:
    return Settings()
```

Минимальные env-переменные:

```env
SERVICE_NAME=menu-service
ENVIRONMENT=local
LOG_LEVEL=INFO
LOG_FORMAT=readable
DATABASE_URL=postgresql+asyncpg://postgres:postgres@localhost:5432/menu_service
```

Для Docker:

```env
DATABASE_URL=postgresql+asyncpg://postgres:postgres@postgres:5432/menu_service
LOG_FORMAT=json
```

## 7. FastAPI app

Создать `app/main.py`.

Приложение должно:

```text
1. Загружать settings.
2. Настраивать logging через dk-common.
3. Подключать CorrelationIdMiddleware.
4. Подключать /health.
5. Подключать routers:
   - menu_items
   - recipes
   - availability
```

Пример:

```python
from fastapi import FastAPI

from dk_common.correlation import CorrelationIdMiddleware
from dk_common.health import build_health_response
from dk_common.logging import configure_logging

from app.core.config import get_settings
from app.api.routes import menu_items, recipes, availability


def create_app() -> FastAPI:
    settings = get_settings()

    configure_logging(
        service_name=settings.service_name,
        environment=settings.environment,
        log_level=settings.log_level,
        log_format=settings.log_format,
    )

    app = FastAPI(title="Menu Service", version="0.1.0")

    app.add_middleware(CorrelationIdMiddleware)

    @app.get("/health")
    async def health():
        return build_health_response(
            service_name=settings.service_name,
            environment=settings.environment,
            version="0.1.0",
        )

    app.include_router(menu_items.router)
    app.include_router(recipes.router)
    app.include_router(availability.router)

    return app


app = create_app()
```

Если в текущей реализации `dk-common` есть `create_health_router()`, можно использовать его. Если нет — использовать `build_health_response()`.

## 8. База данных

### 8.1. PostgreSQL

Menu Service владеет своей схемой/таблицами.

Даже если физически используется один PostgreSQL для всех сервисов, Menu Service не должен читать/писать таблицы других сервисов.

### 8.2. SQLAlchemy async session

Создать `app/db/session.py`.

Ожидаемо:

```python
from collections.abc import AsyncGenerator

from sqlalchemy.ext.asyncio import AsyncSession, async_sessionmaker, create_async_engine

from app.core.config import get_settings

settings = get_settings()

engine = create_async_engine(settings.database_url, echo=False)
AsyncSessionLocal = async_sessionmaker(
    engine,
    expire_on_commit=False,
    class_=AsyncSession,
)


async def get_db_session() -> AsyncGenerator[AsyncSession, None]:
    async with AsyncSessionLocal() as session:
        yield session
```

### 8.3. Declarative base

Создать `app/db/base.py`.

```python
from sqlalchemy.orm import DeclarativeBase


class Base(DeclarativeBase):
    pass
```

Все модели должны наследоваться от этого `Base`.

## 9. Модель данных

### 9.1. `menu_items`

Назначение: хранит блюда.

Поля:

```text
id          UUID primary key
name        string, required
description string, nullable
status      string, required, default active
created_at  timestamp with timezone
updated_at  timestamp with timezone
```

Допустимые значения `status`:

```text
active
disabled
```

Минимальные ограничения:

```text
name not null
status not null
```

Рекомендуемое ограничение:

```text
unique lower(name)
```

Если unique lower(name) усложняет миграцию, можно сделать обычный unique index по `name`, но лучше не допускать дублей с одинаковым именем.

### 9.2. `kitchen_menu_availability`

Назначение: хранит доступность блюда на конкретной кухне.

Поля:

```text
kitchen_id     UUID, required
menu_item_id   UUID, required, foreign key to menu_items.id
is_available   boolean, required, default true
created_at     timestamp with timezone
updated_at     timestamp with timezone
```

Primary key или unique constraint:

```text
UNIQUE(kitchen_id, menu_item_id)
```

Важно:

```text
kitchen_id не является foreign key на kitchens,
потому что kitchens принадлежат Kitchen Service.
```

### 9.3. `recipe_steps`

Назначение: хранит шаги рецепта блюда.

Поля:

```text
id                UUID primary key
menu_item_id      UUID, required, foreign key to menu_items.id
station_type      string, required
operation         string, required
duration_seconds  integer, required
step_order        integer, required
created_at        timestamp with timezone
updated_at        timestamp with timezone
```

Ограничения:

```text
duration_seconds > 0
step_order > 0
UNIQUE(menu_item_id, step_order)
```

Рекомендуемый индекс:

```text
INDEX(menu_item_id, step_order)
```

Допустимые `station_type` на этом этапе можно хранить строкой:

```text
grill
fryer
drinks
packaging
```

Не обязательно создавать enum на уровне БД, но Pydantic-схемы должны валидировать допустимые значения.

## 10. Alembic

### 10.1. Требование

Настроить Alembic внутри `services/menu-service`.

Команды должны работать из директории сервиса:

```bash
cd services/menu-service
poetry run alembic revision --autogenerate -m "create menu service tables"
poetry run alembic upgrade head
```

### 10.2. `alembic/env.py`

`env.py` должен импортировать metadata из `app.db.base.Base`.

Важно: импортировать все модели перед `target_metadata`, чтобы autogenerate видел таблицы.

Пример:

```python
from app.db.base import Base

from app.models.menu_item import MenuItem
from app.models.kitchen_menu_availability import KitchenMenuAvailability
from app.models.recipe_step import RecipeStep

target_metadata = Base.metadata
```

`DATABASE_URL` должен читаться из settings.

Если `DATABASE_URL` async:

```text
postgresql+asyncpg://...
```

Для Alembic может потребоваться заменить драйвер на sync:

```python
database_url = settings.database_url.replace("postgresql+asyncpg://", "postgresql+psycopg://")
```

Если используешь sync Alembic, добавь соответствующий драйвер в зависимости:

```toml
psycopg = { version = "^3.1.0", extras = ["binary"] }
```

Альтернатива — настроить async Alembic template. Для MVP проще sync Alembic + `psycopg`.

### 10.3. Ожидаемая первая миграция

Первая миграция должна создать:

```text
menu_items
kitchen_menu_availability
recipe_steps
```

И constraints:

```text
PK для menu_items
PK для recipe_steps
FK recipe_steps.menu_item_id -> menu_items.id
FK kitchen_menu_availability.menu_item_id -> menu_items.id
UNIQUE(kitchen_id, menu_item_id)
UNIQUE(menu_item_id, step_order)
CHECK duration_seconds > 0
CHECK step_order > 0
```

## 11. API endpoints

### 11.1. Menu items

#### `POST /menu-items`

Создает блюдо.

Request:

```json
{
  "name": "Burger",
  "description": "Classic beef burger",
  "status": "active"
}
```

Response `201 Created`:

```json
{
  "id": "3b675b7e-d4e1-4fc5-80dc-5f3ef89d55ce",
  "name": "Burger",
  "description": "Classic beef burger",
  "status": "active",
  "created_at": "2026-04-30T10:00:00Z",
  "updated_at": "2026-04-30T10:00:00Z"
}
```

Validation:

```text
name required
name non-empty
status in active/disabled
```

Если блюдо с таким именем уже существует:

```http
409 Conflict
```

```json
{
  "error": "menu_item_already_exists",
  "message": "Menu item with this name already exists"
}
```

#### `GET /menu-items`

Возвращает список блюд.

Query params:

```text
status optional: active | disabled
limit optional, default 100
offset optional, default 0
```

Response:

```json
[
  {
    "id": "3b675b7e-d4e1-4fc5-80dc-5f3ef89d55ce",
    "name": "Burger",
    "description": "Classic beef burger",
    "status": "active",
    "created_at": "2026-04-30T10:00:00Z",
    "updated_at": "2026-04-30T10:00:00Z"
  }
]
```

#### `GET /menu-items/{menu_item_id}`

Возвращает блюдо по id.

Если не найдено:

```http
404 Not Found
```

```json
{
  "error": "menu_item_not_found",
  "message": "Menu item not found"
}
```

### 11.2. Recipe steps

#### `POST /menu-items/{menu_item_id}/recipe-steps`

Добавляет шаг рецепта к блюду.

Request:

```json
{
  "station_type": "grill",
  "operation": "cook_patty",
  "duration_seconds": 480,
  "step_order": 1
}
```

Response `201 Created`:

```json
{
  "id": "af2fbaf0-6d10-4d13-b09a-4bb0474e8141",
  "menu_item_id": "3b675b7e-d4e1-4fc5-80dc-5f3ef89d55ce",
  "station_type": "grill",
  "operation": "cook_patty",
  "duration_seconds": 480,
  "step_order": 1,
  "created_at": "2026-04-30T10:05:00Z",
  "updated_at": "2026-04-30T10:05:00Z"
}
```

Validation:

```text
menu_item_id must exist
station_type in grill/fryer/drinks/packaging
operation required, non-empty
duration_seconds > 0
step_order > 0
step_order unique per menu_item_id
```

Ошибки:

```http
404 Not Found
```

```json
{
  "error": "menu_item_not_found",
  "message": "Menu item not found"
}
```

```http
409 Conflict
```

```json
{
  "error": "recipe_step_order_already_exists",
  "message": "Recipe step with this step_order already exists for menu item"
}
```

#### `GET /menu-items/{menu_item_id}/recipe`

Возвращает рецепт блюда.

Response:

```json
{
  "menu_item_id": "3b675b7e-d4e1-4fc5-80dc-5f3ef89d55ce",
  "steps": [
    {
      "id": "af2fbaf0-6d10-4d13-b09a-4bb0474e8141",
      "station_type": "grill",
      "operation": "cook_patty",
      "duration_seconds": 480,
      "step_order": 1
    },
    {
      "id": "d9821310-5982-46f6-8385-6f28f559fb8b",
      "station_type": "packaging",
      "operation": "pack_burger",
      "duration_seconds": 60,
      "step_order": 2
    }
  ]
}
```

Критически важно:

```text
steps должны всегда возвращаться в порядке step_order ASC.
```

Если блюдо не найдено:

```http
404 Not Found
```

### 11.3. Kitchen menu availability

#### `POST /kitchens/{kitchen_id}/menu-items/{menu_item_id}/availability`

Создает или обновляет доступность блюда на кухне.

Request:

```json
{
  "is_available": true
}
```

Response:

```json
{
  "kitchen_id": "d53b7d88-b23c-4bb8-a403-6238c810092a",
  "menu_item_id": "3b675b7e-d4e1-4fc5-80dc-5f3ef89d55ce",
  "is_available": true,
  "created_at": "2026-04-30T10:10:00Z",
  "updated_at": "2026-04-30T10:10:00Z"
}
```

Поведение:

```text
1. Проверить, что menu_item_id существует.
2. Если записи для (kitchen_id, menu_item_id) нет — создать.
3. Если запись есть — обновить is_available.
4. Не проверять kitchen_id через Kitchen Service на этом этапе.
```

Если блюдо не найдено:

```http
404 Not Found
```

```json
{
  "error": "menu_item_not_found",
  "message": "Menu item not found"
}
```

#### `GET /kitchens/{kitchen_id}/menu`

Возвращает меню конкретной кухни.

По умолчанию возвращать только доступные блюда.

Query params:

```text
include_unavailable optional bool, default false
```

Response:

```json
[
  {
    "id": "3b675b7e-d4e1-4fc5-80dc-5f3ef89d55ce",
    "name": "Burger",
    "description": "Classic beef burger",
    "status": "active",
    "is_available": true
  }
]
```

Правила:

```text
1. Если include_unavailable=false:
   - вернуть только блюда, у которых есть availability запись для kitchen_id с is_available=true;
   - вернуть только menu_items.status=active.

2. Если include_unavailable=true:
   - вернуть блюда, у которых есть availability запись для kitchen_id;
   - включить is_available=false;
   - disabled блюда можно вернуть только если это явно удобно для диагностики, но лучше по умолчанию фильтровать active.
```

Для MVP достаточно:

```text
GET /kitchens/{kitchen_id}/menu возвращает active блюда с is_available=true.
```

## 12. Ошибки API

Ошибки должны быть стабильными и машиночитаемыми.

Формат:

```json
{
  "error": "error_code",
  "message": "Human readable message",
  "details": {}
}
```

Минимальные error codes:

```text
menu_item_not_found
menu_item_already_exists
recipe_step_order_already_exists
invalid_station_type
validation_error
internal_error
```

Можно использовать `DKCommonError` из `dk-common`, если он уже реализован удобно.

Если нет — можно сделать локальные exception handlers в `menu-service`, но не добавлять бизнес-ошибки в `dk-common`.

Важно:

```text
dk-common может содержать только инфраструктурные ошибки.
Ошибки вида menu_item_not_found должны жить в menu-service.
```

## 17. Примеры curl

```bash
curl -X POST http://localhost:8000/menu-items \
  -H "Content-Type: application/json" \
  -d '{

    "status": "active"
  }'
```

```bash
curl -X POST http://localhost:8000/menu-items/{menu_item_id}/recipe-steps \
```

```bash
curl http://localhost:8000/menu-items/{menu_item_id}/recipe
```

## 19. Краткая команда для агента

Реализуй `services/menu-service` для этапа 3.

Нужны FastAPI, PostgreSQL, SQLAlchemy async, Alembic, модели `menu_items`, `kitchen_menu_availability`, `recipe_steps`, endpoints меню/рецептов/availability и тесты.

Используй `dk-common` как Poetry path dependency:

```toml
dk-common = { path = "../../libs/python/dk-common", develop = true }
```

Не используй `sys.path.append`.

Не добавляй KDS, Redis, Fulfillment logic, Mongo events и Prometheus business metrics на этом этапе.

Сохраняй recipe steps и всегда возвращай их в порядке `step_order ASC`.
